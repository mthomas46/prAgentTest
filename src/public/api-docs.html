API Documentation

Welcome to the API documentation. This page provides detailed information about all available endpoints, request/response formats, and examples.

Base URL

All API endpoints are prefixed with:

http://localhost:8090/api

Authentication

Currently, the API does not require authentication. All endpoints are publicly accessible.

Response Format

All API responses follow a consistent format:

{
    "data": {}, // The actual response data
    "meta": {
        "timestamp": "2024-04-02T08:00:00.000Z",
        "requestId": "unique-request-id"
    }
}

Error Handling

Error responses follow this format:

{
    "error": "Error Type",
    "message": "Human-readable error message",
    "statusCode": 400,
    "details": {} // Optional additional error details
}

Available Endpoints

Health Check

GET /api/health

Check the health status of the API.

Response

{
    "status": "ok",
    "timestamp": "2024-04-02T08:00:00.000Z",
    "uptime": 3600,
    "memoryUsage": {
        "heapUsed": 123456789,
        "heapTotal": 987654321,
        "external": 123456789
    }
}

Detailed Health Check

GET /api/health/detailed

Get detailed health information including memory usage statistics.

Response

{
    "status": "ok",
    "timestamp": "2024-04-02T08:00:00.000Z",
    "uptime": 3600,
    "memoryUsage": {
        "heapUsed": 123456789,
        "heapTotal": 987654321,
        "external": 123456789,
        "rss": 123456789
    },
    "process": {
        "pid": 12345,
        "version": "v18.0.0",
        "platform": "darwin",
        "arch": "x64"
    }
}

Documentation

GET /api/docs/*

Access various documentation formats:

  • /api/docs/docs - Overview documentation
  • /api/docs/api - API reference
  • /api/docs/health - Health check documentation

All documentation endpoints support both Markdown and HTML responses based on the Accept header.

Project Documentation

GET /api/project-docs/*

Access project documentation files:

  • /api/project-docs/README - Project overview
  • /api/project-docs/CHANGELOG - Version history
  • /api/project-docs/DIALOGUE_LOG - Development dialogue log
  • /api/project-docs/STATISTICS - Project statistics

Interactive Documentation

GET /swagger

Access the interactive Swagger UI documentation:

  • Swagger UI - Interactive API documentation and testing
  • OpenAPI Specification - Raw OpenAPI/Swagger specification in JSON format

Rate Limiting

The API implements rate limiting to ensure fair usage:

  • 100 requests per 15 minutes per IP address
  • Rate limit headers included in responses:
    • X-RateLimit-Limit - Maximum requests per window
    • X-RateLimit-Remaining - Remaining requests in current window
    • X-RateLimit-Reset - Time when the rate limit window resets
